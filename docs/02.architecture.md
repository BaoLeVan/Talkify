# 02 — Cấu trúc Module (Modular Monolith)

## 2.1 Triết lý thiết kế

**Modular Monolith** — một ứng dụng Spring Boot duy nhất, nhưng code được tổ chức thành **các module tách biệt rõ ràng** theo nghiệp vụ. Mỗi module có package riêng, không truy cập trực tiếp vào internal của module khác.

```
Nguyên tắc:
  ✅ Mỗi module = một package boundary độc lập
  ✅ Giao tiếp giữa modules qua: Service interface hoặc Spring ApplicationEvent
  ✅ Không import entity/repository của module khác trực tiếp
  ✅ Shared code đặt trong chat-common
  ❌ Không dùng Aggregate Root, Value Object, Domain Event (DDD patterns)
  ❌ Không có Repository Port/Interface tách biệt
```

**Giao tiếp giữa modules:**
- `ApplicationEventPublisher` (Spring) — in-process, không-blocking, `@Async`.
- Ví dụ: `MessageService` publish `MessageSentApplicationEvent`, `NotificationService` lắng nghe.
- Khi cần tách Microservices sau này: thay `ApplicationEventPublisher` bằng Kafka publisher — không đụng vào business logic.

## 2.4 Simple Layered Architecture — Mỗi module

```
┌─────────────────────────────────────────────────────┐
│                    MODULE (ví dụ: messaging)         │
├─────────────────────────────────────────────────────┤
│  Controller Layer (REST / WebSocket)                 │
│  - Nhận request, validate @Valid, gọi Service        │
│  - Không chứa business logic                         │
├─────────────────────────────────────────────────────┤
│  Service Layer                                       │
│  - Chứa toàn bộ business logic                       │
│  - Gọi Repository để đọc/ghi data                    │
│  - Publish ApplicationEvent khi cần notify module khác│
├─────────────────────────────────────────────────────┤
│  Repository Layer                                    │
│  - Spring Data JPA (extends JpaRepository)           │
│  - Spring Data MongoDB (extends MongoRepository)     │
│  - Custom @Query khi cần                             │
├─────────────────────────────────────────────────────┤
│  Entity / Document Layer                             │
│  - @Entity cho PostgreSQL (JPA)                      │
│  - @Document cho MongoDB                             │
└─────────────────────────────────────────────────────┘
```

## 2.5 Cross-module Communication

### Cách 1: Service phụ thuộc Service khác (cùng Maven module)

```java
// chat-messaging: MessageService cần lấy user info → gọi UserService
@Service
@RequiredArgsConstructor
public class MessageService {

    private final UserService userService;     // inject cross-module service

    public MessageDto sendMessage(Long senderId, SendMessageRequest req) {
        // Kiểm tra sender có tồn tại không
        UserProfileDto sender = userService.getProfileById(senderId);
        // ... business logic
    }
}
```

### Cách 2: Spring ApplicationEvent (loose coupling, async)

```java
// chat-messaging: publish event sau khi lưu message
@Service
@RequiredArgsConstructor
public class MessageService {

    private final ApplicationEventPublisher eventPublisher;

    public MessageDto sendMessage(Long senderId, SendMessageRequest req) {
        // ... lưu message
        MessageDocument saved = messageMongoRepository.save(doc);

        // Notify các module khác — không cần biết ai lắng nghe
        eventPublisher.publishEvent(new MessageSentEvent(
            saved.getId(), saved.getConversationId(), senderId, saved.getContent()
        ));

        return toDto(saved);
    }
}

// chat-notification: lắng nghe và xử lý bất đồng bộ
@Component
@RequiredArgsConstructor
public class MessageEventListener {

    private final PushNotificationService pushService;

    @EventListener
    @Async                   // không block thread của MessageService
    public void onMessageSent(MessageSentEvent event) {
        pushService.notifyOfflineMembers(event.conversationId(), event.content());
    }
}

// ApplicationEvent — simple record, KHÔNG là DomainEvent
public record MessageSentEvent(
    String messageId,
    String conversationId,
    Long senderId,
    String content
) {}
```

## 2.6 Business Rules — Đặt ở Service Layer

Không có Aggregate Root, business rules được đặt trong Service:

```java
@Service
@RequiredArgsConstructor
@Transactional
public class MessageService {

    private final MessageMongoRepository messageRepo;
    private final ConversationJpaRepository convRepo;
    private final RedisTemplate<String, String> redis;
    private final ApplicationEventPublisher eventPublisher;

    public MessageDto sendMessage(Long senderId, String conversationId,
                                  SendMessageRequest req) {
        // ── Rule 1: Người gửi phải là participant ──────────────────────────
        ConversationEntity conv = convRepo.findById(conversationId)
            .orElseThrow(() -> new NotFoundException("Conversation không tồn tại"));

        boolean isParticipant = conv.getParticipants().stream()
            .anyMatch(p -> p.getUserId().equals(senderId) && p.isActive());
        if (!isParticipant) {
            throw new AppException(ErrorCode.NOT_CONVERSATION_MEMBER);
        }

        // ── Rule 2: Nội dung không được rỗng ──────────────────────────────
        if (req.getText() == null || req.getText().isBlank()) {
            throw new AppException(ErrorCode.MESSAGE_CONTENT_EMPTY);
        }

        // ── Rule 3: Idempotency — chống duplicate send ─────────────────────
        String dedupKey = "dedup:msg:" + req.getClientMessageId();
        Boolean isNew = redis.opsForValue()
            .setIfAbsent(dedupKey, "1", Duration.ofHours(24));
        if (Boolean.FALSE.equals(isNew)) {
            // Đã xử lý rồi — trả về message cũ
            return messageRepo.findByClientMessageId(req.getClientMessageId())
                .map(this::toDto)
                .orElseThrow();
        }

        // ── Rule 4: Sequence number (monotonic per conversation) ───────────
        Long seq = redis.opsForValue()
            .increment("seq:" + conversationId);

        // ── Persist ────────────────────────────────────────────────────────
        MessageDocument doc = new MessageDocument();
        doc.setId(String.valueOf(SnowflakeIdGenerator.nextId()));
        doc.setConversationId(conversationId);
        doc.setSenderId(senderId);
        doc.setContent(req.getText());
        doc.setClientMessageId(req.getClientMessageId());
        doc.setSequenceNumber(seq);
        doc.setStatus(MessageStatus.SENT);
        doc.setCreatedAt(Instant.now());
        messageRepo.save(doc);

        // Update last message trên conversation
        convRepo.updateLastMessage(conversationId, doc.getContent(), doc.getCreatedAt());

        // Notify
        eventPublisher.publishEvent(new MessageSentEvent(
            doc.getId(), conversationId, senderId, doc.getContent()
        ));

        return toDto(doc);
    }

    public void editMessage(Long editorId, String messageId, String newText) {
        MessageDocument msg = messageRepo.findById(messageId)
            .orElseThrow(() -> new NotFoundException("Message không tồn tại"));

        // ── Rule: chỉ sender mới được edit ────────────────────────────────
        if (!msg.getSenderId().equals(editorId)) {
            throw new AppException(ErrorCode.FORBIDDEN_EDIT_MESSAGE);
        }

        // ── Rule: không edit message đã xóa ───────────────────────────────
        if (msg.getDeletedAt() != null) {
            throw new AppException(ErrorCode.CANNOT_EDIT_DELETED_MESSAGE);
        }

        // ── Rule: chỉ edit trong 24 giờ ───────────────────────────────────
        if (Duration.between(msg.getCreatedAt(), Instant.now()).toHours() > 24) {
            throw new AppException(ErrorCode.MESSAGE_EDIT_EXPIRED);
        }

        msg.setContent(newText);
        msg.setEditedAt(Instant.now());
        messageRepo.save(msg);
    }

    public void deleteMessage(Long deleterId, String messageId) {
        MessageDocument msg = messageRepo.findById(messageId)
            .orElseThrow(() -> new NotFoundException("Message không tồn tại"));

        // ── Rule: chỉ sender mới được xóa (hoặc admin — check riêng) ─────
        if (!msg.getSenderId().equals(deleterId)) {
            throw new AppException(ErrorCode.FORBIDDEN_DELETE_MESSAGE);
        }

        msg.setDeletedAt(Instant.now());
        messageRepo.save(msg);
    }
}
```
