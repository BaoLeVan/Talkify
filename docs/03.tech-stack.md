## 3.2 Chiến lược Dual-Database

### Tại sao PostgreSQL + MongoDB?

| Tiêu chí | PostgreSQL | MongoDB |
|----------|-----------|---------|
| Dùng cho | User, Group, Conversation, Participant metadata | Messages, Delivery Receipts, Pending Queue |
| Điểm mạnh | ACID, JOINs, complex queries, constraints | High write throughput, flexible schema, native array |
| Write scale | ~10K–50K writes/s (với tuning) | ~100K–500K+ writes/s (sharded) |
| Query lịch sử | ❌ Slow với hàng tỷ rows | ✅ Range query theo `sequenceNumber`, TTL index |
| Schema | Structured, strict | Flexible, document-oriented |
| Khi cần | Conversation list, unread count, role check | Load tin nhắn, search, delivery tracking |

### MongoDB — Vai trò thay thế ScyllaDB
MongoDB được chọn thay ScyllaDB vì:
- **Driver Java (Spring Data MongoDB)** hoàn chỉnh, document phong phú.
- **Aggregation Pipeline** mạnh hơn CQL cho analytics.
- **Flexible Document** phù hợp với `metadata`, `attachments` biến đổi.
- **Change Streams** có thể dùng thay Kafka trong giai đoạn MVP.
- **Atlas / Docker** dễ cài đặt và vận hành hơn.
- Hiệu năng đủ tốt cho hàng trăm triệu messages với sharding đúng strategy.

## 3.3 MongoDB Schema (Message Store)

### Collection: `messages`

```javascript
// Index Strategy:
// { conversationId: 1, sequenceNumber: -1 }   → Load history (cursor-based)
// { conversationId: 1, createdAt: -1 }         → Load by time range
// { messageId: 1 }                             → Lookup by ID (unique)
// { senderId: 1, createdAt: -1 }               → Load sent messages
// { "content.text": "text" }                   → Full-text search

{
  _id: NumberLong("7281946281"),        // Snowflake ID (= messageId)

  sequenceNumber: NumberLong(1501),
  senderId: NumberLong(1001),

  content: {
    text: "Xin chào!",
    attachments: [],
    metadata: {
      "linkPreview": "...",

    }
  },
  replyToMessageId: null,
  isDeleted: false,
  editedAt: null,
```

### MongoDB Initialization Script

```javascript
// Xem docker/mongodb/init-mongo.js
db = db.getSiblingDB('chat_db');

// === Messages ===
db.createCollection('messages');
db.messages.createIndex({ conversationId: 1, sequenceNumber: -1 });
db.messages.createIndex({ conversationId: 1, createdAt: -1 });
db.messages.createIndex({ messageId: 1 }, { unique: true });

// === Delivery Receipts ===
db.createCollection('delivery_receipts');
db.delivery_receipts.createIndex({ messageId: 1, recipientId: 1 }, { unique: true });
db.delivery_receipts.createIndex({ recipientId: 1, readAt: 1 });

// === Pending Messages (TTL 30 ngày) ===
db.createCollection('pending_messages');
db.pending_messages.createIndex({ userId: 1, createdAt: 1 });
db.pending_messages.createIndex(
  { createdAt: 1 },
  { expireAfterSeconds: 2592000 }
);
```

## 3.4 PostgreSQL Schema (Metadata)

```sql
-- Xem docker/postgres/init.sql cho full schema

-- Users
CREATE TABLE users (
    id              BIGINT PRIMARY KEY,
    username        VARCHAR(50) UNIQUE NOT NULL,
    display_name    VARCHAR(100) NOT NULL,
    avatar_url      VARCHAR(500),
    phone_number    VARCHAR(20) UNIQUE,
    email           VARCHAR(255) UNIQUE,
    password_hash   VARCHAR(255) NOT NULL,
    status          VARCHAR(20) DEFAULT 'ACTIVE',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Conversations (metadata only, messages in MongoDB)
CREATE TABLE conversations (
    id                  VARCHAR(36) PRIMARY KEY,
    type                VARCHAR(10) NOT NULL,        -- DIRECT | GROUP
    title               VARCHAR(200),
    avatar_url          VARCHAR(500),
    created_by          BIGINT REFERENCES users(id),
    last_message_id     BIGINT,
    last_message_preview VARCHAR(200),
    last_message_at     TIMESTAMPTZ,
    sequence_counter    BIGINT DEFAULT 0,            -- Atomic via SELECT FOR UPDATE
    created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Participants
CREATE TABLE conversation_participants (
    conversation_id     VARCHAR(36) REFERENCES conversations(id),
    user_id             BIGINT REFERENCES users(id),
    role                VARCHAR(20) DEFAULT 'MEMBER',
    nickname            VARCHAR(100),
    muted_until         TIMESTAMPTZ,
    last_read_sequence  BIGINT DEFAULT 0,
    unread_count        INT DEFAULT 0,
    joined_at           TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    left_at             TIMESTAMPTZ,
    PRIMARY KEY (conversation_id, user_id)
);

-- Group Settings
CREATE TABLE group_settings (
    conversation_id     VARCHAR(36) PRIMARY KEY REFERENCES conversations(id),
    max_members         INT DEFAULT 1000,
    only_admins_send    BOOLEAN DEFAULT FALSE,
    join_approval       BOOLEAN DEFAULT FALSE,
    invite_link         VARCHAR(100) UNIQUE,
    invite_link_expires TIMESTAMPTZ,
    description         TEXT
);
```

```
┌─────────────────────────────────────────┐
│  1. WebSocket Session Registry          │
│     HASH ws:sessions                    │
│     Field: {userId} → {instanceId}      │
│     TTL: N/A (xóa khi disconnect)       │
├─────────────────────────────────────────┤
│  2. User Presence                       │
│     HASH presence:{userId}              │
│     Fields: status, lastSeen, device    │
│     TTL: 300s (auto expire → OFFLINE)   │
├─────────────────────────────────────────┤
│  3. Conversation List Cache             │
│     ZSET user:conv:{userId}             │
│     Score: lastMessageAt (ms)           │
│     Member: conversationId              │
│     TTL: 3600s                          │
├─────────────────────────────────────────┤
│  4. Unread Count                        │
│     STRING unread:{userId}:{convId}     │
│     Value: integer counter              │
├─────────────────────────────────────────┤
│  5. Sequence Number                     │
│     STRING seq:{conversationId}         │
│     INCR atomic → per-conversation seq  │
├─────────────────────────────────────────┤
│  6. Typing Indicator                    │
│     STRING typing:{convId}:{userId} = 1 │
│     TTL: 5s (auto expire)              │
├─────────────────────────────────────────┤
│  7. Rate Limiting                       │
│     STRING ratelimit:{userId}:sec/min   │
│     INCR + EXPIRE                       │
├─────────────────────────────────────────┤
│  8. Pub/Sub Channels                   │
│     ws:delivery:{instanceId}            │
│     → Cross-instance WebSocket routing  │
│     presence:updates                    │
│     → Broadcast presence changes        │
├─────────────────────────────────────────┤
│  9. Idempotency Keys                    │
│     STRING dedup:msg:{clientMsgId}      │
│     TTL: 86400s (24h)                   │
└─────────────────────────────────────────┘
```
